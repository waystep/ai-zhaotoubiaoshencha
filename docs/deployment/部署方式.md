# 智能投标预审智能体 部署方式文档

## 1. 部署概述

### 1.1 系统要求

| 项目 | 要求 |
|------|------|
| **Node.js** | 20.x 或更高版本 |
| **PostgreSQL** | 15.x 或更高版本 |
| **内存** | 最低 4GB，推荐 8GB+ |
| **存储** | 最低 10GB，文档存储需更多 |

### 1.2 支持的部署方式

| 方式 | 适用场景 | 复杂度 |
|------|---------|--------|
| **本地开发** | 开发调试 | 低 |
| **Vercel** | 生产部署（推荐） | 低 |
| **Netlify** | GitHub 连接后的在线演示/生产部署 | 低 |
| **Railway** | 全托管部署 | 低 |
| **Docker** | 自托管部署 | 中 |
| **自托管** | 企业私有化 | 高 |

## 2. 本地开发环境

### 2.1 安装步骤

#### Step 1: 克隆项目

```bash
git clone <repository-url>
cd ai-shencha
```

#### Step 2: 安装依赖

```bash
npm install
```

#### Step 3: 配置环境变量

```bash
# 复制示例配置
cp .env.example .env

# 编辑配置文件
nano .env
```

**必需配置**：
```bash

AUTH_URL=http://localhost:3000
```

```bash

# 执行迁移
npm run db:migrate

# 或直接推送 schema
npm run db:push
```

#### Step 5: 启动开发服务器

```bash

```

访问 http://localhost:3000

## 3. MinerU 服务部署

### 3.1 MinerU 简介

MinerU 是文档解析服务，支持 PDF/Word/Excel/PPT 等格式的解析。

### 3.2 本地部署 (推荐)

```bash
# 使用 Docker 部署 MinerU
docker run -d \
  --name mineru \
  -p 8000:8000 \
  -v /path/to/models:/models \
  opendatalab/mineru:latest

# 或使用 pip 安装
pip install magic-pdf
magic-pdf --start-server --port 8000
```

### 3.3 配置

```bash
# MinerU API 地址
MINERU_API_URL=http://127.0.0.1:8000

# MinerU 后端类型
MINERU_BACKEND=hybrid-auto-engine

# 请求超时（秒）
MINERU_TIMEOUT=300
```

### 3.4 后端类型选择

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| `pipeline` | 通用，支持多语言 | 多语言文档 |
| `vlm-auto-engine` | 本地高精度，仅中英文 | 中文招标文件 |
| `hybrid-auto-engine` | 新一代高精度 | 推荐 |
| `hybrid-http-client` | 远程计算高精度 | 云部署 |

## 4. 数据库部署

### 4.1 本地 PostgreSQL

```bash
# Ubuntu/Debian
sudo apt install postgresql
sudo systemctl start postgresql

# 创建数据库
sudo -u postgres createuser -P looma_user
sudo -u postgres createdb -O looma_user smart_tender_review
```

### 4.2 Neon (云数据库推荐)

1. 访问 https://neon.tech
2. 创建项目
3. 获取连接字符串

```bash
DATABASE_URL=postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/smart_tender_review?sslmode=require
```

### 4.3 Supabase

1. 访问 https://supabase.com
2. 创建项目
3. 获取数据库连接

```bash
DATABASE_URL=postgresql://postgres:password@db.xxx.supabase.co:5432/postgres
```

### 4.4 数据库迁移

```bash
# 生成迁移文件
npm run db:generate

# 执行迁移（生产环境）
npm run db:migrate

# 打开 Studio 查看
npm run db:studio
```

## 5. Vercel 部署 (推荐)

### 5.1 部署步骤

#### Step 1: 连接 GitHub

1. 登录 https://vercel.com
2. Import Project → 选择 GitHub Repository
3. 配置项目

#### Step 2: 配置环境变量

在 Vercel Dashboard → Settings → Environment Variables：

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | PostgreSQL 连接 |
| `AUTH_SECRET` | JWT 密钥 |
| `AUTH_URL` | 应用 URL (Vercel 自动) |
| `ALIBABA_API_KEY` | AI 模型密钥 |
| `MINERU_API_URL` | MinerU 服务地址 |
| `NEXT_PUBLIC_APP_URL` | 前端 URL |

#### Step 3: 部署

Vercel 自动部署，每次 git push 自动触发。

### 5.2 注意事项

- MinerU 需单独部署或使用云服务
- 文件上传需配置持久化存储（如 S3）
- 生产环境关闭 `ENABLE_CRON_IN_DEV`

## 6. Netlify 部署

### 6.1 部署步骤

项目根目录已提供 `netlify.toml`：

```toml
[build]
  command = "npm run build"
  publish = ".next"

[build.environment]
  NODE_VERSION = "20"
```

部署流程：

```bash
npx netlify status
npx netlify link --git-remote-url git@github.com:waystep/ai-zhaotoubiaoshencha.git
npx netlify deploy --prod
```

如果没有现成 Netlify 站点，可使用 `npx netlify init` 创建，并选择 GitHub 仓库作为持续部署来源。

### 6.2 必需环境变量

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | PostgreSQL 连接字符串，推荐 Neon/Supabase 等托管数据库 |
| `AUTH_SECRET` | NextAuth 密钥 |
| `AUTH_URL` | Netlify 线上站点 URL |
| `NEXT_PUBLIC_APP_URL` | Netlify 线上站点 URL |
| `ALIBABA_API_KEY` 或 `ALIBABA_CODING_PLAN_API_KEY` | AI 模型密钥 |
| `MINERU_API_URL` | MinerU 解析服务地址，线上环境不能使用本机 `127.0.0.1` |

### 6.3 演示管理员账号

部署数据库迁移完成后执行：

```bash
npm run db:seed
```

默认账号：

| 项 | 值 |
|----|----|
| 邮箱 | `admin@ai-shencha.local` |
| 密码 | `Admin@123456` |

正式公开演示前建议通过 `SEED_ADMIN_EMAIL` 和 `SEED_ADMIN_PASSWORD` 覆盖默认值，并在 seed 后妥善保存测试账号。

## 8. Docker 部署

### 8.1 Dockerfile (需创建)

```dockerfile
# Dockerfile
FROM node:20-alpine AS builder

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

FROM node:20-alpine AS runner

WORKDIR /app

COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public

ENV NODE_ENV=production
ENV PORT=3000

EXPOSE 3000

CMD ["node", "server.js"]
```

### 8.2 docker-compose.yml (需创建)

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://postgres:postgres@db:5432/smart_tender_review
      - AUTH_SECRET=${AUTH_SECRET}
      - ALIBABA_API_KEY=${ALIBABA_API_KEY}
    depends_on:
      - db

  db:
    image: postgres:15
    environment:
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=postgres
      - POSTGRES_DB=smart_tender_review
    volumes:
      - postgres_data:/var/lib/postgresql/data

  mineru:
    image: opendatalab/mineru:latest
    ports:
      - "8000:8000"
    volumes:
      - mineru_models:/models

volumes:
  postgres_data:
  mineru_models:
```

### 8.3 启动 Docker

```bash
# 构建并启动
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 9. 自托管部署

### 9.1 生产构建

```bash
# 构建
npm run build

# 启动
npm run start
```

### 9.2 PM2 进程管理

```bash
# 安装 PM2
npm install -g pm2

# 启动应用
pm2 start npm --name "looma" -- run start

# 启动 Worker
pm2 start npm --name "looma-worker" -- run worker

# 查看状态
pm2 status

# 保存配置
pm2 save

# 设置开机启动
pm2 startup
```

### 9.3 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 10. 后台 Worker 部署

Worker 用于定时检查文档解析状态。

### 10.1 启动 Worker

```bash

```

### 10.2 配置

```bash
# 检查间隔（秒）
CRON_INTERVAL_SECONDS=30
```

### 10.3 生产部署

Worker 需要作为独立进程运行：

```bash
# PM2

# 在 docker-compose.yml 中添加 worker 服务
```
