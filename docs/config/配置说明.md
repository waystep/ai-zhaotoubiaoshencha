## 2. 环境变量详解

### 2.1 数据库配置

```bash
# PostgreSQL 连接字符串
DATABASE_URL=postgresql://user:password@localhost:5432/smart_tender_review
```

| 参数 | 说明 |
|------|------|
| `user` | 数据库用户名 |
| `password` | 数据库密码 |
| `localhost` | 数据库主机 |
| `5432` | 数据库端口 |
| `smart_tender_review` | 数据库名 |

**示例**：
- 本地开发：`postgresql://postgres:postgres@localhost:5432/smart_tender_review`
- Neon 云数据库：`postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/smart_tender_review?sslmode=require`
- Supabase：`postgresql://postgres:password@db.xxx.supabase.co:5432/postgres`

### 2.2 NextAuth 认证配置

```bash
# 认证密钥（用于 JWT 签名，必须设置）
AUTH_SECRET=your-auth-secret-here

# 认证 URL（生产环境必须设置）
AUTH_URL=http://localhost:3000
```

| 变量 | 说明 | 必需 |
|------|------|------|
| `AUTH_SECRET` | JWT 签名密钥，建议 32+ 字符 | ✅ |
| `AUTH_URL` | 应用 URL，OAuth 回调需要 | ✅ (生产) |

**生成 AUTH_SECRET**：
```bash
# Linux/Mac
openssl rand -base64 32

# Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

### 2.3 OAuth Provider 配置

```bash
# GitHub OAuth
AUTH_GITHUB_ID=your-github-client-id
AUTH_GITHUB_SECRET=your-github-client-secret

# Google OAuth
AUTH_GOOGLE_ID=your-google-client-id
AUTH_GOOGLE_SECRET=your-google-client-secret
```

**GitHub OAuth 配置步骤**：
1. 访问 https://github.com/settings/developers
2. 创建 OAuth App
3. 设置 Authorization callback URL: `http://localhost:3000/api/auth/callback/github`
4. 复制 Client ID 和 Client Secret

**Google OAuth 配置步骤**：
1. 访问 https://console.cloud.google.com/apis/credentials
2. 创建 OAuth 2.0 Client ID
3. 设置 Authorized redirect URI: `http://localhost:3000/api/auth/callback/google`
4. 复制 Client ID 和 Client Secret

### 2.5 MinerU 文档解析配置

```bash
# MinerU API 地址（本地部署后的地址）
MINERU_API_URL=http://127.0.0.1:8000

# MinerU API Key（可选，部分部署需要认证）
MINERU_API_KEY=

# MinerU 请求超时时间（秒）
MINERU_TIMEOUT=300

# MinerU 解析后端类型
MINERU_BACKEND=hybrid-auto-engine
```

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `MINERU_API_URL` | MinerU API 服务地址 | `http://127.0.0.1:8000` |
| `MINERU_API_KEY` | API 认证密钥 | 空 |
| `MINERU_TIMEOUT` | 请求超时时间（秒） | 300 |
| `MINERU_BACKEND` | 解析后端类型 | `hybrid-auto-engine` |

**MinerU 后端类型说明**：

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| `pipeline` | 通用，支持多语言，无幻觉 | 多语言文档 |
| `vlm-auto-engine` | 本地高精度，仅支持中英文 | 中文招标文件 |
| `hybrid-auto-engine` | 新一代本地高精度，支持多语言 | 推荐 |
| `hybrid-http-client` | 远程计算高精度 | 云服务部署 |
| `vlm-http-client` | 远程计算高精度 (VLM) | 云服务部署 |

## 3. Next.js 配置详解

### 3.1 next.config.ts

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  experimental: {
    serverActions: {
      bodySizeLimit: '1gb',  // Server Actions 文件上传限制
    },
  },
  webpack: (config) => {
    // 禁用 canvas 原生模块（react-pdf 兼容）
    config.resolve.alias = {
      ...config.resolve.alias,
      canvas: false,
    };
    return config;
  },
  typescript: {
    ignoreBuildErrors: false,  // 构建时 TypeScript 检查
  },
  eslint: {
    ignoreDuringBuilds: false, // 构建时 ESLint 检查
  },
};

export default nextConfig;
```

### 3.2 配置项说明

| 配置项 | 说明 |
|--------|------|
| `serverActions.bodySizeLimit` | Server Actions 请求体大小限制，支持大文件上传 |
| `webpack.resolve.alias.canvas` | 禁用原生 canvas，react-pdf 服务端兼容 |
| `typescript.ignoreBuildErrors` | 是否忽略 TypeScript 构建错误 |
| `eslint.ignoreDuringBuilds` | 是否忽略 ESLint 构建错误 |

## 4. Drizzle ORM 配置详解

### 4.1 drizzle.config.ts

```typescript
import { defineConfig } from "drizzle-kit";

export default defineConfig({
  schema: "./src/lib/db/schema.ts",  // Schema 文件路径
  out: "./drizzle",                  // 迁移输出目录

    url: process.env.DATABASE_URL || "postgresql://localhost:5432/smart_tender_review",

  verbose: true,                     // 详细日志
  strict: true,                      // 严格模式
```

### 4.2 配置项说明

| 配置项 | 说明 |
|--------|------|

| `dialect` | 数据库类型 (postgresql/mysql/sqlite) |
| `dbCredentials.url` | 数据库连接字符串 |

## 10. 配置问题排查

### 10.1 常见问题

**数据库连接失败**：
- 检查 `DATABASE_URL` 格式
- 确认 PostgreSQL 服务已启动
- 检查网络连接和防火墙

**认证失败**：
- 检查 `AUTH_SECRET` 已设置
- 生产环境检查 `AUTH_URL` 正确
- OAuth 检查回调 URL 配置

**MinerU 解析失败**：
- 检查 MinerU 服务已启动
- 确认 `MINERU_API_URL` 正确
- 检查 `MINERU_TIMEOUT` 设置

**AI 模型调用失败**：
- 检查 `ALIBABA_API_KEY` 有效
- 确认模型名称正确
- 检查 API 配额和限流
