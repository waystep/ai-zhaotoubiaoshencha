# v0.2.0 版本开发功能清单

> 版本目标：从 0.1.0（基础预审）升级到 0.2.0（深度解析 + 模板编制 + 预审增强）
> 编制日期：2026-05-28
> 基线版本：0.1.0

---

## 版本目标概述

| 维度 | v0.1.0（当前） | v0.2.0（目标） |
|------|---------------|---------------|
| 招标文件解析 | 仅提取5类审查项（extractionItems） | 深度拆解为 reviewItems + responseItems，覆盖资格/技术/商务/评分全维度 |
| 投标文件编制 | 不支持 | 基于招标文件拆解结果，自动生成投标文件大纲和模板 |
| 废标预审 | 基础5项检查 | 扩展至15+检测维度，覆盖签字盖章、有效期、保证金等 |
| 报告能力 | 基础报告生成 | 增加问题统计可视化、报告导出 |
| 借鉴竞品 | — | 北京筑龙（智能清标）、e招智胜（137检测点）、标桥（标书检查） |

## 一、招标文件深度解析与拆解（核心）

### 1.1 升级提取 Agent — 启用 reviewItems 表

> **现状**：extraction-agent 仅写入 `extractionItems` 表（title/checkpoint/blocks 三字段），数据库中更完整的 `reviewItems` 表（含 itemType、itemNo、description、location、requirements、consequence、legalReference 等字段）尚未使用。

| 编号 | 功能 | 技术方案 | 涉及文件 |
|------|------|---------|---------|
| **V2-101** | 新建 `reviewItemStorageTool` | 仿照 `extractionItemStorageTool`，写入 `reviewItems` 表的完整字段结构 | `src/mastra/tools/review-item-storage-tool.ts`（新建） |
| **V2-102** | 新建 `responseItemStorageTool` | 写入 `responseItems` 表，提取招标文件中要求投标人响应的内容 | `src/mastra/tools/response-item-storage-tool.ts`（新建） |
| **V2-103** | 重构 extraction-agent 指令 | 从仅5类审查项扩展为多维度提取，同时写入 reviewItems + responseItems | `src/mastra/agents/extraction-agent.ts`（修改） |
| **V2-104** | 新增"资格性要求"提取维度 | 提取资质等级、营业执照、安全生产许可证、人员资格等要求 | extraction-agent 指令 + reviewItems（itemType: "qualification"） |
| **V2-105** | 新增"符合性要求"提取维度 | 提取投标函、授权书、投标保证金、签字盖章等格式性要求 | extraction-agent 指令 + reviewItems（itemType: "compliance"） |
| **V2-106** | 新增"商务条款"提取维度 | 提取付款方式、违约条款、质保期、价格调整机制等 | extraction-agent 指令 + reviewItems（itemType: "commercial"） |

**`bidTemplates` 表结构设计：**

```
bidTemplates
├── id (uuid, PK)
├── projectId (uuid, FK → tenderProjects)
├── tenderDocId (uuid, FK → documents)      -- 关联的招标文件
├── templateName (varchar)                   -- 模板名称
├── templateType (varchar)                   -- technical/commercial/qualification
├── outline (jsonb)                          -- 结构化大纲
│   ├── [{ sectionNo, title, parentId, required, description,
│          linkedReviewItems[], linkedResponseItems[],
│          suggestedContent, minWordCount }]
├── status (enum: draft/generating/completed)
├── generatedBy (varchar)                    -- 生成来源
├── createdAt / updatedAt
```

## 九、验收标准

### 招标文件解析
- [ ] 上传招标文件后，自动提取 **12个维度** 的审查项和响应项
- [ ] 提取结果分别写入 `reviewItems` 和 `responseItems` 表
- [ ] 前端可查看结构化分析结果（章节、关键信息、评分标准）
- [ ] 评标办法自动识别并标注

### 投标文件模板
- [ ] 基于招标文件解析结果，一键生成投标文件大纲
- [ ] 每个章节关联对应的招标要求（reviewItems / responseItems）
- [ ] 标注评分点和权重
- [ ] 可导出 Word 格式的投标文件大纲

### 废标预审
- [ ] 新增6个废标检测维度全部通过测试
- [ ] 每个问题附带具体修改建议
- [ ] 废标概率评估数值与人工判断一致率 ≥ 80%
- [ ] 问题可按类型/严重程度筛选

### 报告
- [ ] 报告包含问题统计图表（饼图/柱状图）
- [ ] 报告可导出 PDF 和 Word 格式
- [ ] 导出的报告格式规范、可直接用于工作汇报
