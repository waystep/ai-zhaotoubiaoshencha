Worker 每 30 秒检查 processing 状态的文档：

```typescript
// src/lib/tasks/document-status-checker.ts
检查文档解析状态：
- 查询 parseStatus = "processing" 的文档
- 调用 MinerU API 查询任务状态
```

```bash

# 配置检查间隔
```

## 5. 备份恢复

### 5.1 数据库备份

```bash
# PostgreSQL 备份
pg_dump $DATABASE_URL > backup_$(date +%Y%m%d).sql

# 压缩备份
gzip backup_$(date +%Y%m%d).sql

# 定时备份脚本 (cron)
0 2 * * * pg_dump $DATABASE_URL | gzip > /backups/looma_$(date +\%Y\%m\%d).sql.gz
```

### 5.2 文件存储备份

```bash
# uploads 目录备份
tar -czf uploads_backup_$(date +%Y%m%d).tar.gz uploads/

# 定时备份脚本
0 2 * * * tar -czf /backups/uploads_$(date +\%Y\%m\%d).tar.gz uploads/
```

### 5.3 恢复流程

```bash
# 数据库恢复
gunzip backup_20240101.sql.gz
psql $DATABASE_URL < backup_20240101.sql

# 文件恢复
tar -xzf uploads_backup_20240101.tar.gz
```

## 7. 故障处理指南

### 7.1 常见故障类型

| 故障 | 现象 | 可能原因 |
|------|------|---------|
| **应用无响应** | 页面加载超时 | 内存溢出、进程崩溃 |
| **数据库连接失败** | 500 错误 | PostgreSQL 未启动、连接数满 |
| **文档解析失败** | parseStatus = failed | MinerU 服务异常、文件损坏 |
| **AI 审查失败** | report.status = failed | API Key 无效、模型限流 |
| **文件上传失败** | 上传错误 | 存储空间满、权限问题 |

### 7.2 排查流程

```
Step 1: 检查应用状态
        pm2 status
        ↓
Step 2: 查看日志
        pm2 logs looma
        ↓
Step 3: 检查数据库
        psql $DATABASE_URL -c "SELECT 1"
        ↓
Step 4: 检查 MinerU
        curl http://127.0.0.1:8000/health
        ↓
Step 5: 检查环境变量
        printenv | grep DATABASE_URL
        printenv | grep ALIBABA_API_KEY
        ↓
Step 6: 重启服务
        pm2 restart looma
```

### 7.3 故障处理预案

#### 应用无响应

```bash
# 检查进程
pm2 status

# 重启应用
pm2 restart looma

# 如果进程不存在，重新启动
pm2 start npm --name "looma" -- run start
```

#### 数据库连接失败

```bash
# 检查 PostgreSQL
systemctl status postgresql

# 启动 PostgreSQL
systemctl start postgresql

# 检查连接数
psql $DATABASE_URL -c "SELECT count(*) FROM pg_stat_activity"

# 重启应用
pm2 restart looma
```

#### MinerU 解析失败

```bash
# 检查 MinerU 服务
docker ps | grep mineru
curl http://127.0.0.1:8000/health

# 重启 MinerU
docker restart mineru

# 查看失败文档
npm run db:studio
# 查找 parseStatus = "failed" 的文档
```

#### AI 审查失败

```bash
# 检查 API Key
curl -H "Authorization: Bearer $ALIBABA_API_KEY" https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation

# 检查报告状态
npm run db:studio
# 查找 status = "failed" 的报告

# 重试审查
# 通过前端界面重新发起审查
```

## 9. 运维脚本模板

### 9.1 每日检查脚本

```bash
#!/bin/bash
# daily_check.sh

echo "=== 智能招标审查平台 每日运维检查 ==="
echo "时间: $(date)"

# 检查应用
pm2 status

# 检查数据库连接
psql $DATABASE_URL -c "SELECT 1" && echo "数据库: OK" || echo "数据库: FAIL"

# 检查 MinerU
curl -s http://127.0.0.1:8000/health && echo "MinerU: OK" || echo "MinerU: FAIL"

# 检查存储空间
df -h | grep -E '/$|uploads'

echo "=== 检查完成 ==="
```

### 9.2 备份脚本

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d)
BACKUP_DIR="/backups"

# 数据库备份
pg_dump $DATABASE_URL | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# 文件备份
tar -czf $BACKUP_DIR/uploads_$DATE.tar.gz uploads/

# 清理 7 天前的备份
find $BACKUP_DIR -name "*.gz" -mtime +7 -delete

echo "备份完成: $DATE"
```
