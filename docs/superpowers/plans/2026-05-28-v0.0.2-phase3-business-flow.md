# v0.0.2 Phase 3: Core Business Flow (A1-A7) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现 7 个预设智能体的核心业务逻辑——招标文件解析（含法规自动验证）、投标文件生成、投标文件解析、投标预审（含规则集驱动）、风险定位、法规解析、分析报告，以及投标文件在线编辑器。

**Architecture:** 在 Phase 1 的模型管理 + Phase 2 的知识库基础上，重构 Mastra Agent 系统。每个智能体从 agentDefinitions 表读取配置（模型、知识库、规则集、参数），通过 AgentConfigService 动态创建 Mastra Agent 实例。保留 v0.1.0 的 Supervisor 协调模式，扩展为新流程编排。

**Tech Stack:** Mastra AI Framework, Next.js 15, Drizzle ORM, pgvector

**Spec:** `docs/superpowers/specs/2026-05-28-v0.0.2-platform-design.md` Section 三、四

**Depends on:** Phase 1（模型绑定）、Phase 2（知识库 + 规则集）

- [ ] **Step 1: 实现 AgentConfigResolver**

核心：运行时根据 agentKey 动态解析智能体的完整配置（模型、知识库、规则、参数）。

```typescript
export class AgentConfigResolver {
  // 解析智能体当前使用的模型配置（Endpoint、Key、Model ID）
  async resolveModel(agentKey: string, orgId: string): Promise<ModelConfig>;

  // 解析智能体关联的知识库列表
  async resolveKnowledgeBases(agentKey: string, orgId: string): Promise<KnowledgeBase[]>;

  // 解析智能体绑定的规则集
  async resolveRuleSet(agentKey: string, orgId: string): Promise<RuleItem[]>;

  // 解析完整配置
  async resolveAll(agentKey: string, orgId: string): Promise<AgentFullConfig>;
}
```

`model-resolver.ts`：从 `agentModelBindings` 表读取当前首选模型，支持回退到备选模型。
`knowledge-retriever.ts`：从知识库关联配置读取，返回知识库列表 + 搜索方法。
`rule-loader.ts`：从 `ruleSets` → `ruleItems` 读取启用的规则列表。
