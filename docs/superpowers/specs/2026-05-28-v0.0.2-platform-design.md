# v0.0.2 AI 智能招投标工具平台 — 业务设计规格书

> 版本：v0.0.2
> 日期：2026-05-28
> 状态：已确认

---

## 一、平台定位

v0.0.2 从"单一预审工具"升级为**可配置的招投标 AI 工具平台**：

- 预设 7 个智能体，用户可直接选用
- 支持调整智能体参数（模型选择、知识库关联、审查规则）
- 支持本地大模型 + 云端 API 混合部署，节约企业成本
- 所有智能体独立可调用，也可在项目空间内串联为完整工作流
- 输出物支持 JSON 结构化，可对接飞书、译筑等低代码平台

## 二、架构总览

### 2.1 三层架构

```
┌─────────────────────────────────────────────────────┐
│              业务层（用户交互）                        │
│  项目管理 │ 招标解析 │ 投标生成 │ 投标预审 │ 报告     │
├─────────────────────────────────────────────────────┤
│              智能体层（核心能力）                      │
│  A1 招标解析 │ A2 投标生成 │ A3 投标预审 │ A4 风险定位 │
│  A5 法规解析 │ A6 分析报告 │ A7 投标文件解析          │
├─────────────────────────────────────────────────────┤
│              输出适配层（结构化 + 集成）               │
│  JSON Schema │ REST API │ Webhook │ 模板引擎          │
├─────────────────────────────────────────────────────┤
│              基础设施层（平台能力）                    │
│  模型管理 │ 知识库管理 │ 用量监控 │ 用户/租户管理      │
└─────────────────────────────────────────────────────┘
```

### 2.2 七大模块

| 模块 | 职责 | 面向角色 |
|------|------|---------|
| M1 模型管理 | 配置本地/云端大模型，查看调用统计 | 管理员 |
| M2 知识库管理 | 分类知识库（法律法规/企业模板/风险项），上传/编辑/向量化 | 管理员/高级用户 |
| M3 智能体管理 | 展示预设智能体，配置模型、知识库、参数、规则 | 管理员 |
| M4 招标文件解析 | 上传招标文件 → 结构化解析 + 法规自动验证 | 业务用户 |
| M5 投标文件生成 | 基于解析结果 + 知识库 → 生成投标文件 v1.0 样稿 | 业务用户 |
| M6 投标预审 | 上传投标文件 → 多维度风险检测 → 报告输出 | 业务用户 |
| M7 集成输出 | JSON API + Webhook，对接飞书/译筑等外部平台 | 管理员/开发者 |

## 三、核心业务流程

### 3.1 流程一：招标文件 → 生成投标文件

```
用户上传招标文件
       │
       ▼
  A1 招标文件解析
  ├── 按规则拆解：章节、条款、评分标准、资质要求、关键参数
  ├── 自动扫描法规引用（正则匹配《xxx法》、国标GB/T等）
  ├── 发现法规引用 → 触发 A5 法律法规解析
  │   ├── 检索法律法规库最新条款
  │   ├── 验证引用是否为最新版本
  │   └── 标注：✅已验证 / ⚠️版本已更新 / ❌未收录
  └── 输出：结构化要点 + 法规验证标注
       │
       ▼
  A2 投标文件生成
  ├── 读取：招标要点 + 法规验证 + 企业模板库 + 法律法规库
  └── 输出：投标文件 v1.0 样稿（结构化章节 + 内容）
       │
       ▼
  用户二次编辑
  ├── 在线编辑（平台内置编辑器）
  └── 导出 Word → 本地修改 → 重新上传
```

### 3.2 流程二：用户上传投标文件 → 预审

```
用户上传投标文件（有或无关联招标文件均可）
       │
       ▼
  A7 投标文件解析（结构化：章节、内容、关键信息提取）
       │
       ├── 有关联招标文件？
       │   └── Yes → A1 解析结果作为对照基准
       │             A3 投标预审 → 逐项比对投标文件是否满足招标要点
       │
       ├── 投标文件中发现法规引用？
       │   └── Yes → A5 法律法规解析 → 验证是否匹配最新条款
       │
       ▼
  A3 投标预审（综合输出）
  ├── 招标要点满足度（有招标文件时）
  ├── 法规条款匹配度
  └── 风险项列表（高/中/低等级）
       │
       ├── A4 风险定位 → 页/块级精准定位坐标
       ├── A5 法规解析（补充详细法律依据）
       │
       ▼
  A6 分析报告输出
  ├── 结构化预审报告（评分、统计、建议）
  ├── 导出预审报告（PDF/Word）
  ├── 导出风险清单（Excel）
  └── JSON API 输出（对接外部平台）
```

### 3.3 独立性保证

每个智能体可单独调用，不强制走完整流程：

- 用户可以只做招标解析（A1），不生成投标文件
- 用户可以只上传投标文件做预审（A7→A3），无需先解析招标文件
- 用户可以只做风险定位（A4），针对已有的风险列表

## 四、预设智能体详细设计

### 4.1 智能体清单

| ID | 名称 | 输入 | 核心逻辑 | 输出 | 关联知识库 |
|----|------|------|---------|------|-----------|
| A1 | 招标文件解析 | 招标文件 PDF/Word | 按规则拆解章节/条款/评分/资质；发现法规引用自动触发 A5 验证 | 结构化要点 + 法规验证标注 | 法律法规库 |
| A2 | 投标文件生成 | A1 结果 + 知识库 | 结合招标要点、法规验证、企业模板生成投标文件 v1.0 | 投标样稿（结构化章节） | 企业模板库 + 法律法规库 + 自定义库 |
| A3 | 投标预审 | 投标文件 + 招标要点（如有） | 比对招标要点满足度 + 法规匹配度 → 风险检测 | 风险项列表（等级/类型/描述） | 风险项库 + 法律法规库 |
| A4 | 风险定位 | 风险项 + 投标文件 | 在原文中定位风险项位置（页码/段落/块坐标） | 精准定位坐标 | — |
| A5 | 法律法规解析 | 条款引用文本 | 检索法律法规库最新条款，验证引用是否过期/错误 | 最新条款 + 合规分析 | 法律法规库 |
| A6 | 分析报告 | 全部审查结果 | 汇总评分、统计、风险摘要、建议 | 结构化预审报告 | — |
| A7 | 投标文件解析 | 投标文件 PDF/Word | 结构化提取章节、内容、关键信息 | 投标文件结构化数据 | — |

### 4.2 智能体编排关系

```
            ┌─────────────┐
            │  A1 招标解析  │ ← 独立可调用，内部可触发 A5
            └──────┬──────┘
                   │ 解析结果 + 法规验证
                   ▼
            ┌─────────────┐
            │  A2 投标生成  │ ← 依赖 A1 结果
            └──────┬──────┘
                   │ 样稿
                   ▼
              用户编辑
                   │
                   ▼
            ┌─────────────┐
            │  A7 投标解析  │ ← 完全独立
            └──────┬──────┘
                   │
                   ▼
            ┌─────────────┐
            │  A3 投标预审  │ ← 独立可调用，内部可触发 A5
            └──────┬──────┘
                   │ 风险项
              ┌────┴────┐
              ▼         ▼
       ┌──────────┐ ┌──────────┐
       │ A4 风险定位│ │ A5 法规解析│ ← 并行
       └────┬─────┘ └────┬─────┘
            └─────┬──────┘
                  ▼
           ┌────────────┐
           │ A6 分析报告  │
           └────────────┘
```

### 4.3 智能体可配置参数

| 参数 | 说明 | 示例 |
|------|------|------|
| AI 模型 | 从已注册模型列表中选择 | A1 用 Qwen3.6-27B 本地，A2 用 GLM-5.1 云端 |
| 关联知识库 | 指定调用时检索哪些知识库 | A2 关联「企业模板库」+「法律法规库」 |

| Max Tokens | 单次调用最大输出 | A6 报告=8192，A1 解析=4096 |
| 审查规则集 | 加载预定义的审查规则 | A3 加载「建设工程废标检测规则 v2」 |

| 配置项 | 说明 |

| Endpoint | API 地址（云端 URL 或本地 Ollama 地址） |

| 能力标签 | `text` / `vision` / `reasoning` |

## 六、知识库与规则配置

### 6.1 知识库分类

| 知识库 | 内容 | 来源 |
|--------|------|------|
| 法律法规库 | 国家法律、行业法规、地方规章、国标/行标 | 手动上传、自动抓取 |
| 企业模板库 | 投标函模板、技术标模板、商务标模板、资信标模板 | 企业积累、历史标书 |
| 风险项库 | 废标风险项、格式风险项、合规风险项、业绩风险项 | 预审自动沉淀、行业共享 |
| 自定义库 | 企业按需创建（如特定行业知识） | 用户自定义 |

### 6.2 法律法规库结构

| 字段 | 说明 |
|------|------|
| 法规名称 | 如《招标投标法》 |
| 法规类型 | 法律/行政法规/部门规章/地方性法规/国标 |
| 文号 | 如国务院令第613号 |
| 发布日期/生效日期 | 版本追踪 |
| 最新版本标记 | 是否为当前有效版本（A5 验证用） |
| 条款正文 | 分章节结构化存储 |
| 向量索引 | pgvector 向量化，支持语义检索 |

### 6.3 企业模板库结构

| 字段 | 说明 |
|------|------|
| 模板名称 | 如「公路工程施工投标文件模板」 |
| 行业分类 | 建筑/市政/公路/水利/电力... |
| 模板类型 | 技术标/商务标/资信标/投标函/综合标 |
| 章节结构 | JSON 树形（章节标题、排序、必填/选填） |
| 每章节内容框架 | 写作要点、注意事项、参考法规 |
| 来源项目 | 关联历史项目（可选） |

### 6.4 风险项库结构

| 字段 | 说明 |
|------|------|
| 风险类型 | 废标/格式/合规/业绩/资质/技术 |
| 风险描述 | 标准化风险描述文本 |
| 检测规则 | 检测逻辑（关键词/比对/阈值） |
| 法律依据 | 关联法律法规库条款 |
| 严重等级 | 高（废标级）/中/低 |
| 修改建议 | 标准修改建议文本 |
| 出现频次 | 历史预审中出现的次数（自动统计） |

### 6.5 知识库与智能体关联

| 智能体 | 法律法规库 | 企业模板库 | 风险项库 | 自定义库 |
|--------|:---------:|:---------:|:-------:|:-------:|
| A1 招标解析 | ✅ | ❌ | ❌ | ❌ |
| A2 投标生成 | ✅ | ✅ | ❌ | ✅ |
| A3 投标预审 | ✅ | ❌ | ✅ | ❌ |
| A5 法规解析 | ✅ | ❌ | ❌ | ❌ |

### 6.6 审查规则集

管理员可创建和管理审查规则集，分配给 A3 投标预审智能体：

```
规则集
├── 规则集名称：如「建设工程废标检测规则 v2」
├── 适用行业：建筑/市政/公路...
├── 关联智能体：A3 投标预审
├── 规则列表：
│   ├── R001: 投标函必须签字盖章（权重：高，类型：废标）
│   ├── R002: 投标有效期不少于招标文件要求天数（权重：高，类型：废标）
│   ├── R003: 保证金金额与招标文件一致（权重：高，类型：废标）
│   ├── ...
├── 每条规则：
│   ├── 规则编号/名称
│   ├── 检测类型（关键词匹配/数值比对/语义比对/存在性检查）
│   ├── 严重等级（高/中/低）
│   ├── 检测逻辑描述（自然语言 + 参数）
│   └── 启用/禁用开关
```

## 七、A1 招标解析 — 法规自动验证机制

A1 解析过程中自动检测法规引用并触发验证：

```
A1 解析招标文件
  │
  ├─ 提取章节结构
  ├─ 提取审查项（资格/符合/技术/商务）
  ├─ 提取评分标准
  ├─ 提取关键参数（保证金/工期/质保）
  │
  ├─ 扫描法规引用
  │   ├── 正则匹配：《xxx法》《xxx条例》国标GB/T xxx 等
  │   ├── 每个引用 → 检索法律法规库
  │   ├── 找到 → 验证是否为最新版本
  │   │   ├── 是最新 → 标注 ✅ 已验证
  │   │   └── 非最新 → 标注 ⚠️ 引用版本已更新，最新为 xxx
  │   └── 未找到 → 标注 ❌ 法规库中无此条目，需人工确认
  │
  └─ 输出：完整解析结果 + 法规验证标注
```

## 八、集成输出模块（M7）

### 8.1 四种结构化 JSON 输出

| 输出物 | JSON 结构概要 | 消费场景 |
|--------|-------------|---------|
| 招标解析结果 | `{项目信息, 审查项[], 资质要求[], 评分标准[], 关键参数, 章节结构}` | 飞书多维表格、译筑表单 |
| 投标文件样稿 | `{章节树[], 每章节{标题, 内容, 关联要求[], 评分点}, 元数据}` | 飞书文档、译筑文档 |
| 风险清单 | `{风险项[]{等级, 类型, 描述, 定位(页/块), 法律依据, 修改建议}, 统计摘要}` | 飞书表格/任务、译筑工单 |
| 预审报告 | `{评分, 等级, 风险摘要, 分维度统计, 建议汇总, 风险项引用[]}` | 飞书文档/Wiki、译筑报表 |

### 8.2 双向集成机制

- **REST API（拉取）**：`GET /api/v1/projects/:id/outputs/:type`（type = analysis | draft | risks | report）
- **Webhook（推送）**：管理员配置 URL + 触发事件（`analysis.completed` / `review.completed` / `report.completed`），完成时自动 POST JSON
- **JSON Schema 版本化**：`GET /api/v1/schemas/:type` 返回输出结构定义

### 8.3 典型集成场景

| 场景 | 流程 |
|------|------|
| 风险项 → 飞书多维表格 | 预审完成 → Webhook 推送风险清单 JSON → 飞书自动化写入表格 |
| 预审报告 → 飞书文档 | API 获取报告 JSON → 飞书 CLI 创建文档 |
| 招标解析 → 译筑工单 | 解析完成 → 推送 JSON → 译筑创建分工单 |
| 投标样稿 → 译筑审批流 | 样稿生成 → 推送 JSON → 触发审批流程 |

## 十一、账号与登录体系

### 11.1 登录方式矩阵

| 登录方式 | 说明 | 适用场景 |
|---------|------|---------|
| **手机号 + 短信验证码** | 输入手机号 → 发送验证码 → 验证登录 | 首选方式，移动端友好，企业用户最常用 |
| **账号密码** | 用户名 + 密码（管理员分配或自行注册） | 传统企业用户习惯，适合桌面端 |
| **邮箱 + 密码** | 邮箱地址 + 密码 | 跨国/跨区企业，邮箱为主的企业 |
| **飞书扫码/授权登录** | OAuth 2.0 接入飞书，扫码或授权跳转 | 已使用飞书的企业，免注册直接登录 |
| **微信扫码登录** | 微信开放平台 OAuth 2.0 | 移动端用户、微信生态企业 |
| **企业 SSO 单点登录** | SAML 2.0 / OIDC 对接客户现有身份系统 | 大型企业客户，与内部 OA/ERP 等系统账号同步 |

### 11.3 第三方登录对接

| 第三方平台 | 协议 | 对接方式 | 自动同步 |
|-----------|------|---------|---------|
| **飞书** | OAuth 2.0 | 飞书开放平台注册应用 → 获取 client_id/secret → 授权回调 | 首次登录自动创建平台账号，同步姓名/头像/部门 |
| **微信** | OAuth 2.0 | 微信开放平台注册网站应用 → 扫码授权 → 回调 | 首次登录绑定手机号后创建账号 |
| **企业 SSO** | SAML 2.0 / OIDC | 管理员在后台配置 IdP 元数据/回调地址 | 登录时自动同步账号和部门信息 |

### 11.4 企业 SSO 对接流程

```
管理员配置阶段：
  ├── 选择协议：SAML 2.0 / OIDC / LDAP
  ├── 填写 IdP 信息：Entity ID、SSO URL、证书
  ├── 配置字段映射：哪些字段对应用户名/邮箱/部门/角色
  └── 测试连接

用户登录流程：
  ├── 用户选择「企业账号登录」
  ├── 跳转至企业 IdP 登录页
  ├── 用户在企业系统完成认证
  ├── IdP 回传身份信息
  ├── 平台自动匹配/创建用户
  └── 登录成功，进入平台

账号同步规则：
  ├── 首次登录 → 自动创建平台账号 + 关联 SSO 身份
  ├── 后续登录 → 自动更新姓名/部门/角色（按字段映射配置）
  ├── SSO 账号离职/禁用 → 平台账号自动禁用（可选）
  └── 一个平台账号可绑定多个第三方身份（手机号+飞书+SSO 通用一个账号）
```

### 11.5 登录安全策略

| 策略 | 说明 |
|------|------|
| 验证码有效期 | 短信/邮件验证码 5 分钟有效 |
| 登录失败锁定 | 连续 5 次失败锁定账号 15 分钟 |
| 密码强度 | 至少 8 位，含大小写字母+数字 |
| 会话管理 | Token 有效期 24 小时，支持多设备同时在线 |
| 操作日志 | 记录登录时间、IP、设备、方式 |

## 十二、设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 平台定位 | 预设智能体 + 用户可调参数 | 平衡易用性和灵活性 |
| 架构方案 | 工作流驱动（项目空间为载体） | 招投标天然项目制，v0.1.0 已有项目概念可复用 |
| 知识库结构 | 分类多库 | 按类型分库，智能体可按需检索，效率更高 |
| 文档编辑 | 在线编辑 + 导出 Word 两者兼备 | 兼顾便捷性和企业习惯 |
| 模型配置 | 多模型 + 按智能体分配 | 不同任务选最优模型，兼顾质量和成本 |
| 业务流程 | 独立可组合 | 每个智能体可单独使用，灵活度高 |
| 本地模型 | Qwen3.6-27B 统一 | 企业已验证可用，简化运维 |
| 云端模型 | GLM-5.1 + DeepSeek-V4-Pro + GPT-5.5(备选) | 企业已有 API，成本可控 |
