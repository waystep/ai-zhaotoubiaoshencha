# Why: 智能投标预审平台 1.0

> 版本：1.0  
> 日期：2026-05-27  
> 定位：面向招投标与资格预审材料的 AI 辅助预审平台

## 1. 产品面向谁

智能投标预审平台面向需要反复处理招标文件、投标文件、资格预审文件和合规附件的组织用户。1.0 阶段优先服务三类角色：

| 用户画像 | 典型目标 | 痛点 | 平台价值 |
|---|---|---|---|
| 一线审查/业务经办 | 快速形成首版问题清单 | 文件长、版本多、人工对照易疲劳 | 上传后进入解析、审查、定位、报告的一体化流程 |
| 复核/项目负责人 | 把关关键风险和审查口径 | 难以快速判断问题来源和严重程度 | 通过结构化报告和 PDF 定位回到原文复核 |
| 组织管理者 | 观察项目风险分布和工作量 | 过往审查经验难沉淀 | 通过项目、文档、报告、问题统计形成可复盘资产 |

平台的边界也很明确：它不是自动裁决系统，不替代人工审查、法律意见或正式签批；它承担的是预读、对照、归纳、定位和报告草稿生成。

## 2. 场景怎么挖掘出来

1.0 的场景来自对招投标/资格预审工作流的拆解，而不是从技术能力倒推页面。核心观察是：审查人员真正耗时的不是“看一个结论”，而是把多个长文档里的强制性条款、响应内容、表格、签章和附件逐项对齐，并且能向他人解释问题来自哪里。

因此 1.0 没有先做通用聊天机器人，而是把链路压成一个可演示闭环：

1. 项目维度归集材料。
2. 上传并解析文件，保留页码、区块和坐标。
3. 从招标文件侧提取审查项。
4. 通过多智能体审查投标文件或响应文件。
5. 生成结构化问题、摘要、评分和建议。
6. 在报告详情页回到 PDF 原文位置复核。
7. 在统计页沉淀组织级风险分布。

这个链路背后的需求发现方法是“任务链路优先”：先确认用户要完成的业务任务，再判断 AI 应该在哪些环节降低认知负担，最后把每个 AI 结论都设计成可追溯、可复核、可人工接管。

## 3. 当前有多少人在用

截至 1.0 版本，仓库没有接入正式运营埋点，也没有可审计的生产用户统计表。因此不应虚构“已有多少人使用”的数字。

当前可以被验证的是：

| 验证类型 | 1.0 状态 | 说明 |
|---|---|---|
| 功能闭环验证 | 已完成 | 上传、解析、审查会话、报告、PDF 问题定位、统计看板可以形成演示闭环 |
| 内部演示验证 | 已完成 | 适合作为黑客松/投标演示版本展示业务理解和技术路径 |
| 生产使用验证 | 待补充 | 需要线上部署、测试账号、真实样例数据、访问日志和关键事件埋点 |
| 效率量化验证 | 待补充 | 需要记录人工审查耗时、AI 首版问题清单耗时、人工复核通过率、误报/漏报 |

2.0 建议优先补三类数据：账号与组织数、项目/文档/报告创建数、审查结果被人工确认或驳回的比例。这些数据比单纯访问量更能证明平台价值。

## 4. 我的思考方式

这个产品采用“人机协同 + 证据优先 + 规范驱动”的方法论。

| 方法 | 含义 | 在 1.0 中的体现 |
|---|---|---|
| 人机协同 | AI 负责预读、对照、归纳，人负责确认、取舍和责任判断 | 报告是辅助材料，关键结论需要人工复核 |
| 证据优先 | 每个问题尽量回到文档页码、区块和坐标 | `reviewIssues.location` 与 PDF 高亮联动 |
| 结构化沉淀 | 把一次审查变成可查询、可统计、可复盘的数据 | 项目、文档、审查项、报告、问题全部落库 |
| 分阶段落地 | 先做闭环，再做规模化、权限、审计、稳定性 | 1.0 聚焦演示闭环，2.0 再补生产化能力 |
| 规范驱动开发 | 需求、领域模型、接口、验收标准先行 | 后续通过 SDD 文档和 skill 固化迭代流程 |

## 5. 1.0 为什么有价值

1.0 的价值不在于“把 AI 接到系统里”，而在于它把招投标预审拆成了可操作的产品链路：

- 用户不是打开一个空白对话框，而是围绕项目、文档和报告推进工作。
- AI 不是只输出一段自然语言，而是把审查结果写入结构化数据。
- 报告不是孤立结论，而是可以回到 PDF 原文位置核对。
- 平台不是一次性演示页面，而是已经具备组织隔离、项目管理、统计沉淀和后续迭代空间。

这使得 2.0 的升级可以围绕真实业务增强，而不是推倒重写。

## 6. 2.0 前需要确认的关键问题

| 方向 | 需要确认 |
|---|---|
| 线上试用 | Netlify 站点、数据库、环境变量、管理员测试账号 |
| 样例数据 | 是否使用脱敏真实材料，还是构造演示材料 |
| 审查口径 | 不同招标类型的强制性条款、评分规则、否决项如何模板化 |
| 人工反馈 | 问题是否支持确认、驳回、修改、备注和责任人 |
| 数据指标 | 2.0 要用哪些指标证明效率、质量和可控性 |
